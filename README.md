# Java-TS-Gen [JTG]

**Annotation-driven TypeScript files generation library for Java records.**

JTG is a developer-time Maven plugin. You annotate a Java `record` with `@TsRecord`, run one command, and a `.ts` file appears right next to your Java source — no build lifecycle integration required, no config sprawl.

```bash
mvn jtg:generate
```

---

## Quick Start

### 1. Add JTG to your project

In your `pom.xml`:
```xml
<dependencies>
  <!-- The annotation (source-retention: zero runtime overhead) -->
  <dependency>
    <groupId>io.github.loomforge</groupId>
    <artifactId>jtg-annotation</artifactId>
    <version>${version}</version>
    <scope>provided</scope>
    <optional>true</optional> 
  </dependency>
</dependencies>

<build>
  <plugins>
    <plugin>
      <groupId>io.github.loomforge</groupId>
      <artifactId>jtg-maven-plugin</artifactId>
      <version>${version}</version>
    </plugin>
  </plugins>
</build>
```

### 2. Annotate your records

```java
import io.github.loomforge.annotation.TsRecord;

@TsRecord
public record User(String name, int age, boolean active, UUID id) {}
```

### 3. Run

_JTG_ runs automatically during `generate-sources`, so any standard build triggers it:

```bash
mvn compile       # jtg:generate fires automatically
mvn package       # same
mvn jtg:generate  # or invoke it explicitly anytime
```

**Result** — `User.ts` is created in the same directory as `User.java`:

```typescript
// Generated by JTG on 2025-04-05 12:00:00
// Source: User.java
// Do not edit this file manually — changes will be overwritten.

export interface User {
  name: string;
  age: number;
  active: boolean;
  id: string;
}
```

## Annotation Options

```java
@TsRecord(
    exportName = "UserDTO",   // Override the TypeScript identifier (default: record name)
    asType     = true         // Emit `type` alias instead of `interface`
)
public record User(String name, int age) {}
```

**With `asType = true`:**

```typescript
export type UserDTO = {
  name: string;
  age: number;
};
```

## Type Mappings

| Java                                         | TypeScript              |
|----------------------------------------------|-------------------------|
| `byte`, `short`, `int`, `long`               | `number`                |
| `float`, `double`                            | `number`                |
| `Integer`, `Long`, `Double`, `BigDecimal`    | `number`                |
| `boolean`, `Boolean`                         | `boolean`               |
| `char`, `Character`, `String`                | `string`                |
| `UUID`, `LocalDate`, `LocalDateTime`, etc.   | `string`                |
| `List<T>`, `Set<T>`, `Collection<T>`         | `Array<T>`              |
| `Map<K, V>`                                  | `Record<string, V>`     |
| `Optional<T>`                                | `T \| null`             |
| Unknown / user-defined types                 | emitted as-is           |

## Plugin Configuration

JTG runs automatically on `generate-sources` with zero configuration. All parameters below are optional.

```xml
<plugin>
  <groupId>io.github.loomforge</groupId>
  <artifactId>jtg-maven-plugin</artifactId>
  <version>${version}</version>
  <configuration>
    <!-- Extra source dirs to scan (beyond src/main/java) -->
    <additionalSourceDirs>
      <dir>src/api/java</dir>
    </additionalSourceDirs>

    <!-- Print every file scanned to the log -->
    <verbose>true</verbose>
  </configuration>
</plugin>
```

Override the phase if needed (e.g. bind to `process-sources` instead):

```xml
<executions>
  <execution>
    <id>jtg-generate</id>
    <phase>process-sources</phase>
    <goals><goal>generate</goal></goals>
  </execution>
</executions>
```

Skip on demand:

```bash
mvn compile -Djtg.skip=true
```

## Building JTG Locally

```bash
git clone https://github.com/loomforge/java-ts-gen.git
cd java-ts-gen
mvn install
```
